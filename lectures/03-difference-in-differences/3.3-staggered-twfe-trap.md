# 3.3 — The Staggered TWFE Trap

> **Part of:** [Lecture 3 — Difference-in-Differences](./README.md)
> **Read when:** units adopt treatment at *different times* — the single most common way a competent analyst produces a confidently wrong number.

---

## 1. The setup that looks harmless

A PM rolls a feature out in waves: Cohort A in January, Cohort B in March,
Cohort C in June. You have panel data on everyone. The reflexive move —
and it's what almost everyone reaches for — is **two-way fixed effects
(TWFE)**:

```math
Y_{it} = \alpha_i + \lambda_t + \beta D_{it} + \epsilon_{it}
```

Unit fixed effects $\alpha_i$, time fixed effects $\lambda_t$, a single
treatment dummy $D_{it}$ that flips on when unit $i$'s wave arrives. Read
$\beta$ as "the effect." Ship.

**This is wrong, and not subtly.** Under staggered timing with effects
that vary — across cohorts or over time since treatment — $\beta$ can be
biased, and in realistic cases it **flips sign**: a genuinely positive
treatment reports as negative. This section is why. It's the discovery
(Goodman-Bacon 2021, and the literature it launched) that rewrote applied
DiD.

---

## 2. What TWFE is actually computing

Goodman-Bacon's result: that single $\beta$ is not a clean effect. It is a
**weighted average of every possible 2×2 DiD** you can form from the data.
With three cohorts and a never-treated group, TWFE silently runs all of
these and averages them:

| # | Comparison | Clean? |
|---|---|---|
| 1 | Each treated cohort vs. **never-treated** | ✅ Good |
| 2 | Earlier cohort vs. **later cohort, before the later one is treated** | ✅ Good — the later cohort is a valid not-yet-treated control |
| 3 | **Later cohort vs. earlier cohort, after the earlier one is already treated** | ❌ **Forbidden** |

Comparison 3 is the poison. When TWFE evaluates Cohort B against Cohort A
in a period where **A is already treated**, it is using *treated units as
the control group*. A's outcome is no longer a clean $Y(0)$ counterfactual
— it already contains A's treatment effect. Subtracting a contaminated
control injects that effect, with the wrong sign, into B's estimate.

---

## 3. The negative-weighting trap, concretely

Here is the mechanism, in the form it actually bites.

The UK gets a feature in January. It works, and its effect **grows over
time** — revenue climbs steadily for six months (a *dynamic* treatment
effect, the completely normal case). France gets the same feature in March.

Now TWFE forms the forbidden comparison: **France in April vs. the UK in
April**, using the UK as control. But the UK's April revenue is still
climbing from its *January* treatment. So the "control" is rising fast —
and against a rising control, France's perfectly good March launch looks
like a *relative decline*. TWFE records that as a negative contribution.

Goodman-Bacon showed these forbidden comparisons can receive **negative
weights** in the overall average. Combine negative weights with dynamic
effects and the sum can invert the sign of the true effect entirely.

**The bias is worst exactly when the treatment is a growing success** —
which is the situation you most want to measure correctly. A feature whose
effect builds over time (adoption, network effects, habit formation) is
the textbook trigger.

---

## 4. Verified: the sign actually flips

Not a cautionary tale — a simulation (code in §6). 100 units, two cohorts
(early adopts at t=2, late at t=12), 20 periods. The true effect is
**positive and grows** at 0.5 per period since treatment. Averaged over all
treated observations, the **true ATT is +3.98**.

```
  True ATT (average over treated observations):  +3.98
  TWFE beta_D:                                    −1.24
  Bias:                                           −5.22   (−131%)
```

**A treatment with a true effect of +3.98 reports as −1.24.** Not
attenuated — *reversed*. A PM reading that TWFE output concludes the
feature *hurts* revenue and kills a genuine winner.

Push the dynamics steeper (effect grows at 1.0/period, true ATT +7.96) and
TWFE returns **−2.46** — the gap only widens. The bias scales with how
dynamic the effect is, which is the opposite of reassuring.

And the fix works: a clean **not-yet-treated** comparison — the early
cohort's change measured against the late cohort *while the late cohort is
still untreated* — recovers ATT(g=2, t=5) = **2.006** against a true
**2.0**. The truth was always in the data. TWFE just averaged it with
poison.

---

## 5. The fix, stated once

**Never let an already-treated unit serve as a control.** Every modern
estimator (§3.4) is a different implementation of that one rule:

- **Callaway-Sant'Anna:** only ever compare a cohort to the *never-treated*
  or *not-yet-treated*. Explicitly forbids comparison 3.
- **Sun-Abraham:** saturate cohort × relative-time so no cohort's effect
  contaminates another's.
- **Wooldridge ETWFE:** interact cohort × period fully, so OLS is
  mathematically blocked from forming the forbidden average.
- **Borusyak / Gardner:** estimate the counterfactual from untreated cells
  only, then impute.

For diagnosis before you switch: the **Goodman-Bacon decomposition** itself
is a tool. It shows you what fraction of your $\beta$ came from forbidden
comparisons and how much negative weight they carried. If the forbidden
share is tiny (e.g. a large never-treated group and short panels), plain
TWFE may be nearly fine. If it's large, your estimate is built on poison,
and §3.4 is mandatory, not optional.

## 6. Python

```python
import numpy as np
import pandas as pd
import statsmodels.formula.api as smf

# ---- Reproduce the sign flip ----
rng = np.random.default_rng(0)
T = 20
cohorts = {"early": (2, range(50)), "late": (12, range(50, 100))}
rows = []
for name, (g, units) in cohorts.items():
    for i in units:
        ui = rng.normal(0, 0.3)                 # unit fixed effect
        for t in range(T):
            treated = t >= g
            te = 0.5 * (t - g + 1) if treated else 0.0    # DYNAMIC: grows over time
            y = ui + 0.1 * t + te + rng.normal(0, 0.2)
            rows.append(dict(unit=i, t=t, g=g, D=int(treated), Y=y, tau=te))
df = pd.DataFrame(rows)

true_att = df.loc[df.D == 1, "tau"].mean()
twfe = smf.ols("Y ~ C(unit) + C(t) + D", data=df).fit().params["D"]
print(f"true ATT = {true_att:+.2f}   TWFE = {twfe:+.2f}   bias = {twfe-true_att:+.2f}")
# true ATT = +3.98   TWFE = -1.24   bias = -5.22   <-- SIGN FLIP

# ---- A clean not-yet-treated comparison recovers the truth ----
def att_gt(df, g, t, base):
    treat = df[df.g == g]
    d_treat = (treat[treat.t == t].set_index("unit").Y
               - treat[treat.t == base].set_index("unit").Y).mean()
    ctrl = df[df.g > t]                          # NOT-YET-treated at time t
    d_ctrl = (ctrl[ctrl.t == t].set_index("unit").Y
              - ctrl[ctrl.t == base].set_index("unit").Y).mean()
    return d_treat - d_ctrl

print(f"ATT(g=2,t=5) clean = {att_gt(df, 2, 5, base=1):.3f}  (true 2.0)")
# ATT(g=2,t=5) clean = 2.006  (true 2.0)

# ---- Diagnose share of forbidden weight before switching ----
# pip install bacondecomp ; from bacondecomp import bacon
# bacon(df, "Y", "D", id_var="unit", time_var="t")
# Look at the "Later vs Earlier Treated" row -> that's the poison.
```

## 7. Common misreadings

**"TWFE with unit and time fixed effects is the standard, so it's safe."**
Safe with a *single* treatment time. With staggered timing and any effect
heterogeneity it silently forms forbidden comparisons and can flip signs.
§4.

**"My β is positive, so heterogeneity bias isn't a problem for me."**
A positive β under staggered timing might be a *biased-downward* version of
a much larger true effect — or a coincidence of weights. You can't know
without decomposing. A right-looking sign is not a clean estimate.

**"Staggered timing just means more data, which is better."**
Timing variation is what *creates* the forbidden comparisons. It is the
hazard, not extra safety. This is the README §3 lesson: the danger axis is
timing, not sample size.

**"The bias is small, a few percent."**
The verified example is −131% and reverses the sign. The bias scales with
how dynamic the effect is, and growing-over-time effects are the norm, not
the exception. §4.

**"I'll just drop the already-treated periods from the control."**
That instinct is correct — and it *is* what the modern estimators do
systematically. Doing it by hand for one comparison isn't enough; you need
it enforced across every (cohort, time) cell. §3.4.
