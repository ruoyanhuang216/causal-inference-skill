# 8.3 — Inverse Probability Weighting

> **Part of:** [Lecture 8 — Selection on Observables](./README.md)
> **Prerequisite:** [§8.2](./8.2-propensity-score-matching.md) — matching, which IPW improves on by *not* throwing data away.

Matching discards every unmatched control (§8.2 §1). **IPW uses all the data**
by reweighting it into a *pseudo-population* where treatment is independent of
the covariates — as if it had been randomized.

---

## 1. The idea: build a pseudo-population by reweighting

If treated units are *rare* in some region of covariate space, each one stands
in for many similar untreated units — so **up-weight the rare, down-weight the
common.** Weighting each observation by the inverse of its probability of the
treatment it received creates a synthetic sample in which $D \perp X$, and a
simple weighted difference in means becomes causal.

The unit of analysis is the **individual observation.** Every user $i$ gets a
weight $W_i$ from their *actual* treatment $D_i$ and their propensity $e(X_i)$.

---

## 2. The weights — and the ATT-vs-ATE question you asked

The weights differ by *which population you're simulating*, and this is the
crux of your question.

### For the ATE (effect if you treated *everyone* vs. *no one*)

You reweight **both** arms to look like the **whole population**:

```math
W_i = \frac{1}{e(X_i)} \ \text{ if treated}, \qquad W_i = \frac{1}{1 - e(X_i)} \ \text{ if control}
```

A treated user who was *unlikely* to be treated ($e = 0.1$) is rare and gets a
big weight (10); they represent all the similar users who *weren't* treated.

### For the ATT (effect on the *treated* population)

You leave the treated arm **as-is** and reweight only the controls to *resemble
the treated*:

```math
W_i = 1 \ \text{ if treated}, \qquad W_i = \frac{e(X_i)}{1 - e(X_i)} \ \text{ if control}
```

**Why is the treated weight exactly 1 for the ATT but $1/e(X)$ for the ATE?**
Because of *which population you're simulating*:

- **ATE** asks about the *whole* population. Neither arm, as observed, looks
  like the whole population — the treated arm over-represents high-propensity
  users. So you must reweight the treated arm *too*, by $1/e(X)$, to recover the
  full-population average. Both arms get bent toward the population.
- **ATT** asks only about the *treated* population. The treated arm **already
  is** that population — no reweighting needed, so weight = 1. You only need to
  make the *controls* look like the treated, which is what
  $\frac{e(X)}{1-e(X)}$ does (it up-weights controls who *look* treated, i.e.
  have high $e(X)$).

**The rule:** the group that *is already* your target population keeps weight 1;
the other group gets reweighted toward it. For the ATT the treated are the
target; for the ATE both arms are reweighted toward the population.

---

## 3. The estimand at the unit level

The **Horvitz-Thompson** IPW estimator for the ATE:

```math
\hat{\tau}_{\text{ATE}}^{\text{IPW}} = \frac{1}{N}\sum_{i=1}^N \frac{D_i Y_i}{e(X_i)} \; - \; \frac{1}{N}\sum_{i=1}^N \frac{(1 - D_i) Y_i}{1 - e(X_i)}
```

Read it per observation: a treated user contributes $Y_i / e(X_i)$ to the
treated average; a control contributes $Y_i / (1 - e(X_i))$ to the control
average; the difference is the effect.

**A refinement you should always use:** the **Hájek** (self-normalized)
estimator divides each arm by the *sum of its weights* instead of by $N$. It's
still consistent, has lower variance, and is bounded within the range of $Y$ —
the raw Horvitz-Thompson estimator above can wander outside it when weights are
extreme.

---

## 4. The variance danger — the reason §8.4 exists

Look at the denominators. A treated user with $e(X) = 0.01$ gets weight $1/0.01
= 100$; their single outcome is multiplied by 100 and can **dominate the entire
estimate.** IPW is *unbiased* (if the propensity is right) but can have
enormous **variance** — extreme propensity scores near 0 and 1 produce extreme
weights, exploding your standard errors and destroying statistical power.

This is the same overlap problem from
[§8.1 §4](./8.1-propensity-and-unconfoundedness.md#4-overlap-and-the-r--auc-illusion),
now expressed as variance. Two partial fixes:

- **Trimming:** drop or cap observations with $e(X) < 0.1$ or $> 0.9$ (or
  Winsorize the weights). Reduces variance at the cost of changing the estimand
  slightly (you're now estimating for the region of overlap).
- **Stabilized weights:** multiply treated weights by $P(D=1)$ and control
  weights by $P(D=0)$. Same estimand, far tamer weights.

But the real fix — bounding the variance *without* discarding data or changing
the estimand — is **AIPW (§8.4)**, which applies the big weights to *residuals*
instead of raw outcomes.

---

## 5. Python

```python
import numpy as np

# df: D (0/1), Y (outcome), ps (estimated propensity, TRIMMED to overlap)
e = df["ps"].clip(0.05, 0.95)      # trim extreme scores first
D, Y = df["D"].values, df["Y"].values

# ---- ATE weights: reweight BOTH arms to the population ----
w_ate = np.where(D == 1, 1/e, 1/(1-e))

# ---- ATT weights: treated = 1, controls reweighted toward the treated ----
w_att = np.where(D == 1, 1.0, e/(1-e))

# ---- Hajek (self-normalized) ATE: divide by SUM of weights, not N ----
t = (w_ate * D * Y).sum() / (w_ate * D).sum()
c = (w_ate * (1-D) * Y).sum() / (w_ate * (1-D)).sum()
print(f"IPW ATE (Hajek) = {t - c:.3f}")

# ---- Inspect the weights BEFORE trusting the estimate ----
print(f"max control weight = {w_ate[D==0].max():.1f}")   # if this is 100+, you have
                                                          # an overlap problem -> AIPW (8.4)
```

## 6. Common misreadings

**"IPW and matching give the same thing."**
Matching discards unmatched controls and targets the ATT; IPW uses everyone and
can target either ATE or ATT via the weights. §1–2.

**"The treated always get weight $1/e(X)$."**
Only for the **ATE**. For the **ATT** the treated keep weight 1 — they *are* the
target population. §2.

**"IPW is unbiased, so it's safe."**
Unbiased ≠ low variance. One user at $e = 0.99$ can blow up the estimate and the
SEs. Trim, stabilize, or go doubly-robust. §4.

**"Just cap the weights and move on."**
Capping reduces variance but changes the estimand (you're now estimating over
the trimmed region). Say so, and prefer AIPW when you can. §4.

**"Use the Horvitz-Thompson estimator by default."**
Prefer **Hájek** (self-normalized): same consistency, lower variance, stays
within the outcome range. §3.
