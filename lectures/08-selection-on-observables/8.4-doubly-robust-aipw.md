# 8.4 — Doubly Robust Estimation (AIPW)

> **Part of:** [Lecture 8 — Selection on Observables](./README.md)
> **Prerequisite:** [§8.2 Matching](./8.2-propensity-score-matching.md) and [§8.3 IPW](./8.3-inverse-probability-weighting.md) — AIPW is the reconciliation of both.

**AIPW = Augmented Inverse Probability Weighting.** It's the modern default for
selection-on-observables, and it ended a long turf war between two camps:

- **The regression camp:** *"Fit an ML model for the outcome $Y$ from $(X, D)$"*
  (G-computation, T/S-learners).
- **The weighting camp:** *"No, regression extrapolates — use the propensity
  score and IPW"* (§8.3).

AIPW's answer: **do both, and you only have to get one of them right.**

---

## 1. The estimator

Train **three** models: the propensity $\hat{e}(X)$, and two outcome models
$\hat{\mu}_1(X) = E[Y \mid X, D{=}1]$ and $\hat{\mu}_0(X) = E[Y \mid X, D{=}0]$.
The AIPW score for each unit $i$:

```math
\Gamma_i = \underbrace{\big[\hat{\mu}_1(X_i) - \hat{\mu}_0(X_i)\big]}_{\text{regression estimate}} + \underbrace{\frac{D_i\big(Y_i - \hat{\mu}_1(X_i)\big)}{\hat{e}(X_i)}}_{\text{treated IPW correction}} - \underbrace{\frac{(1 - D_i)\big(Y_i - \hat{\mu}_0(X_i)\big)}{1 - \hat{e}(X_i)}}_{\text{control IPW correction}}
```

The ATE is the plain average:

```math
\hat{\tau}_{\text{ATE}}^{\text{AIPW}} = \frac{1}{N}\sum_{i=1}^N \Gamma_i
```

Read the structure: **a regression prediction, corrected by an IPW term applied
to the regression's residuals.** The regression does the heavy lifting; the
propensity score cleans up its bias. The actual treatment $D_i$ acts as an
on/off switch — if $D_i = 1$ the control correction vanishes; if $D_i = 0$ the
treated correction vanishes.

---

## 2. The doubly-robust magic — two chances to be right

The estimate is unbiased if **either** the propensity model **or** the outcome
model is correct (they needn't both be). Trace each case through the formula:

**Perfect propensity, garbage outcome.** The outcome model predicts nonsense, so
the residuals $(Y - \hat\mu)$ are huge. But because $\hat{e}(X)$ is exactly
right, the IPW correction terms reweight those residuals to cancel the
regression's bias precisely. It **collapses to a valid IPW estimator.**

**Garbage propensity, perfect outcome.** The propensity weights are crazy. But
because the outcome model is perfect, $Y_i = \hat{\mu}(X_i)$ exactly, so the
residuals $(Y - \hat\mu)$ are **zero**. The crazy weights are multiplied by
zero and vanish. It **collapses to a valid regression estimator.**

### 2.1 Verified — the 4-cell table

A simulation (code in §5): true ATE **3.0**, with a nonlinear DGP (an
interaction term in both the true propensity and the true outcome). "Correct"
models include the interaction; "wrong" models omit it.

```
  naive difference in means : 4.10   bias +1.10   <- confounded
  AIPW, both models correct : 3.00   bias +0.00
  AIPW, e correct, mu WRONG : 3.01   bias +0.01   <- still unbiased
  AIPW, e WRONG, mu correct : 3.00   bias +0.00   <- still unbiased
  AIPW, BOTH models WRONG   : 3.36   bias +0.36   <- biased (as expected)
```

Exactly the theory: unbiased whenever *at least one* model is right; biased only
when **both** are wrong. Two independent shots at the truth.

## 3. Semiparametric efficiency — the DoorDash probe

Beyond robustness, AIPW hits the **semiparametric efficiency bound** — the
lowest variance any estimator can achieve here — and it does so by *taming the
IPW variance explosion* (§8.3 §4).

**The setup.** DoorDash's opt-in bonus; outcome = deliveries next week. You run
plain **IPW**. One control driver (full-time, highly motivated) looks exactly
like the opt-ins, so XGBoost gives them $e(X) = \mathbf{0.999}$.

**Q1 — what weight does IPW apply, and what does it do?**

```math
W = \frac{1}{1 - e(X)} = \frac{1}{1 - 0.999} = \mathbf{1000}
```

This driver's raw outcome is multiplied by 1000. If they complete 50 deliveries,
IPW feeds $50 \times 1000 = \mathbf{50{,}000}$ deliveries into the control
average. This single outlier inflates the control mean to the moon — making the
\$50 bonus look like a *failure* — and the **standard errors explode**, killing
your power to detect anything.

**Q2 — how does AIPW protect the business?** *"Standard IPW applies massive
weights to massive raw numbers. AIPW applies massive weights only to small
errors."*

1. The **outcome model does the heavy lifting.** This driver is highly
   motivated, and that's in their features $X$, so the regression correctly
   predicts a high baseline — say $\hat{\mu}_0(X) = 48$ deliveries.
2. The **residual is tiny:** $Y - \hat{\mu}_0 = 50 - 48 = \mathbf{2}$.
3. The **correction is bounded:** the AIPW control term is
   $\frac{Y - \hat\mu_0}{1 - e(X)} = \frac{2}{0.001} = \mathbf{2000}$ — not
   50,000.

The driver adds **2,000** units of *correction* instead of **50,000** units of
*raw noise*. The regression already explained their high baseline; the weight
only touches what's left over. *Verified*: across simulations with extreme
propensity scores, **IPW's variance is 18× AIPW's** (§5). That variance
reduction is exactly semiparametric efficiency, and it's why AIPW is the
default.

## 4. Implementing the two outcome models — T-learner vs. S-learner

Your question: *do you train two separate models, and do you score both $\mu_1$
and $\mu_0$ for every observation — even one whose treatment is already known?*

**Yes to both.** Causal inference is a missing-data problem: for a *treated*
user you observe $Y(1)$, but the AIPW score needs $\hat\mu_0(X_i)$ too — their
**hallucinated counterfactual** ("what if this treated user hadn't been
treated?"). So you generate *both* predictions for *every* unit, regardless of
their real $D_i$. Two architectures:

**T-learner (two models).** Split by reality:
1. Model 1: train on `D == 1` rows, predict $Y$ from $X$.
2. Model 0: train on `D == 0` rows, predict $Y$ from $X$.
3. Score your **entire** dataset through *both* → $\hat\mu_1$ and $\hat\mu_0$ for
   everyone.

**S-learner (one model).** Train a single model on all data with $D$ as a
feature. Then score twice, forcing the $D$ column:
- Overwrite $D = 1$ for all rows → predict → $\hat\mu_1$.
- Overwrite $D = 0$ for all rows → predict → $\hat\mu_0$.

Both are valid for AIPW. The T-learner captures treatment-covariate
interactions more freely (separate models); the S-learner is one artifact to
maintain but can regularize the treatment effect toward zero if $D$ is drowned
out by many features. Then the AIPW formula's $D_i$ switch picks which residual
to correct for each unit.

**One caution for production:** if $\hat\mu$ and $\hat{e}$ are fit with flexible
ML on the *same* data you evaluate on, overfitting reintroduces bias. The fix is
**cross-fitting** — fit the nuisances on other folds — which is exactly Double
ML (Lecture 9). AIPW + cross-fitting *is* DML for the ATE.

## 5. Python

```python
import numpy as np, statsmodels.api as sm
addc = lambda M: sm.add_constant(M, has_constant="add")

def aipw_ate(Y, D, X, correct=True):
    """AIPW ATE. `correct` toggles whether the models include the interaction
    term (to demonstrate double robustness)."""
    # --- propensity ---
    Z = np.column_stack([X, X[:,0]*X[:,1]]) if correct else X
    e = np.clip(sm.Logit(D, addc(Z)).fit(disp=0).predict(addc(Z)), .01, .99)
    # --- outcome, S-learner: predict BOTH states for EVERY unit ---
    base = np.column_stack([X, X[:,0]*X[:,1]]) if correct else X[:, :1]
    m = sm.OLS(Y, addc(np.column_stack([base, D]))).fit()
    mu1 = m.predict(addc(np.column_stack([base, np.ones(len(Y))])))
    mu0 = m.predict(addc(np.column_stack([base, np.zeros(len(Y))])))
    # --- doubly-robust score ---
    gamma = (mu1 - mu0) + D*(Y - mu1)/e - (1 - D)*(Y - mu0)/(1 - e)
    return gamma.mean()

# Double robustness: unbiased if EITHER model is right (true ATE = 3.0)
# both correct -> 3.00 ; e-only -> 3.00 ; mu-only -> 3.01 ; both wrong -> 3.36
# Production: econml.dr.DRLearner, doubleml.DoubleMLIRM (adds cross-fitting)
```

## 6. Common misreadings

**"AIPW needs both models correct."**
The opposite is its whole point — unbiased if **either** is correct. Two chances
to be right. §2.

**"For a treated user I only need $\hat\mu_1$."**
You need $\hat\mu_0$ too — their counterfactual. Score both states for every
unit. §4.

**"AIPW removes the overlap problem."**
It *tames the variance* from moderate extreme weights, but a propensity of
*exactly* 0 or 1 still has no counterfactual and $1/e$ still blows up. Overlap
must hold; AIPW just makes near-violations survivable. §3.

**"S-learner and T-learner give the same estimate."**
Both are valid, but the S-learner can shrink the treatment effect toward zero if
$D$ is one weak feature among many; the T-learner fits it freely. §4.

**"AIPW with XGBoost nuisances is fine as-is."**
Not without **cross-fitting** — flexible ML on the evaluation data overfits and
rebiases. Cross-fit the nuisances (that's DML, Lecture 9). §4.

**"Doubly robust means robust to unobserved confounding."**
No — it's robust to *misspecification of the two models*, given
unconfoundedness. An unobserved confounder breaks AIPW exactly as it breaks
everything in this lecture. §2, [§8.1](./8.1-propensity-and-unconfoundedness.md).
