# 6.3 — Matrix Completion

> **Part of:** [Lecture 6 — Synthetic Control & Counterfactual Estimators](./README.md)
> **Prerequisite:** [§6.1](./6.1-classic-scm.md). This is the estimator for when SCM's "one clean treated unit" assumption collapses entirely.

---

## 1. The core idea: causal inference is a missing-data problem

The interview trigger: *"We have 50 cities, they adopt the feature at different
times, and some turn it off and back on. We can't build 50 synthetic controls.
What do we do?"* The answer is **Matrix Completion**.

The reframe (Athey, Bayati, Doudchenko, Imbens & Khosravi 2021) is the
"Netflix" view. A recommender has a **users × movies** matrix, mostly empty
because most users haven't seen most movies. Matrix completion (nuclear-norm
regularization) fills the empties by learning the latent factors of users and
movies.

Panel causal inference is *the same problem*:

```
             time →
          ┌─────────────────────────────┐
   unit   │  Y   Y   Y   ?   ?   ?  ...  │   ? = treated cells (post-adoption)
     ↓    │  Y   Y   Y   Y   Y   ?  ...  │       = the missing counterfactual Y(0)
          │  Y   ?   ?   Y   Y   Y  ...  │       (toggled off, then on again)
          └─────────────────────────────┘
```

- **Rows** = units (cities, merchants, users).
- **Columns** = time periods.
- **Cells** = the outcome (revenue).

Wherever a unit is **treated**, its observed value is *not* the untreated
potential outcome $Y(0)$ — so **treat that cell as missing.** Then complete the
matrix from the untreated cells, and the filled-in values *are* your
counterfactuals.

---

## 2. The method

1. Build the units × time outcome matrix.
2. **Delete** every treated (unit, time) cell — set it to `NaN`. The mask of
   what's treated can be *anything*: staggered, toggling, hundreds of treated
   units.
3. Run a **low-rank matrix completion** (nuclear-norm-regularized SVD) on the
   remaining *untreated* cells, tuning the regularization by cross-validation.
   This learns the latent unit- and time-factors that generate $Y(0)$.
4. The algorithm imputes the deleted cells: $\hat{Y}_{it}(0)$.
5. The effect is observed minus imputed on the treated cells:

```math
\hat{\tau}_{it} = Y_{it} - \hat{Y}_{it}(0), \qquad \widehat{\text{ATT}} = \frac{1}{|\text{treated}|}\sum_{(i,t)\,\text{treated}} \hat{\tau}_{it}
```

The low-rank assumption is the identifying one: $Y(0)$ is governed by a small
number of latent factors (unit types × time patterns), so the untreated cells
carry enough structure to reconstruct the missing ones. It's the factor-model
assumption of SCM (§6.1 §1), made explicit and estimated globally rather than
one-treated-unit-at-a-time.

*Verified* (code in §4): a rank-3 panel, 60 units × 30 periods, with
**staggered *and* toggling** treatment (some units adopt at different times,
some switch back off), true effect **2.0**. Deleting the 504 treated cells and
soft-imputing recovers $\hat\tau = \mathbf{2.05}$.

---

## 3. Why it's a staff-level superpower

**1. It handles utter chaos.** Classic SCM needs one treated unit and a clean
pre-period. Callaway-Sant'Anna ([Lecture 3 §3.4](../03-difference-in-differences/3.3-staggered-did.md))
needs *absorbing* treatment. Matrix completion needs **neither**:

| Rollout pattern | SCM | Callaway-Sant'Anna | Matrix Completion |
|---|---|---|---|
| One treated unit, clean pre | ✅ | — | ✅ |
| Staggered, absorbing | ~ | ✅ | ✅ |
| **Toggling on/off** | ❌ | ❌ | ✅ |
| **Hundreds of treated units** | ❌ | ✅ | ✅ |

Any treated period is just "missing," so the rollout schedule is irrelevant.
This is the same **non-absorbing** capability that made de Chaisemartin-
D'Haultfœuille special in DiD ([Lecture 3 §3.3 §7.4](../03-difference-in-differences/3.3-staggered-did.md))
— matrix completion is the SCM-family answer to the same messiness.

**2. It scales.** Matrix factorization is one of the most optimized operations
in all of tech (it powers every recommender). You can complete a
millions-of-users × thousands-of-days matrix far faster than you can solve a
convex-hull optimization per treated unit.

## 4. Python

```python
import numpy as np

def matrix_complete(Y_obs, treated_mask, rank_penalty, n_iter=200):
    """Y_obs: (N,T) observed outcomes. treated_mask: (N,T) bool, True = treated.
    Soft-impute (iterative SVD with nuclear-norm soft-thresholding)."""
    untreated = ~treated_mask
    fill = np.where(treated_mask, Y_obs[untreated].mean(), Y_obs)   # init
    for _ in range(n_iter):
        U, s, Vt = np.linalg.svd(fill, full_matrices=False)
        s = np.maximum(s - rank_penalty, 0)          # nuclear-norm soft threshold
        low_rank = (U * s) @ Vt
        fill = np.where(untreated, Y_obs, low_rank)  # keep observed, impute treated
    return fill                                       # imputed Y(0) in treated cells

Y0_hat = matrix_complete(Y_obs, treated_mask, rank_penalty=8)
tau = (Y_obs[treated_mask] - Y0_hat[treated_mask]).mean()
print(f"ATT = {tau:.3f}")     # 2.05 on the verified staggered+toggling panel (true 2.0)

# Tune rank_penalty by CV: hold out some UNTREATED cells, pick the penalty that
# best predicts them. Production: gsynth (R, Xu 2017), or the MCPanel package.
```

## 5. Common misreadings

**"Matrix completion needs a clean pre-period like SCM."**
No — it only needs *enough untreated cells overall* to learn the low-rank
structure. Treated cells can be scattered anywhere. §1–2.

**"It works for any rollout, so it has no assumptions."**
It has a strong one: $Y(0)$ is **low-rank** (few latent factors). If the
untreated structure can't reconstruct the treated cells, the imputation is
wrong. Tune the rank by cross-validation and check held-out untreated fit. §2.

**"Toggling treatment breaks it like it breaks Callaway-Sant'Anna."**
The opposite — toggling is fine, because on/off just changes *which* cells are
missing. This is matrix completion's headline advantage. §3.

**"Bigger rank always fits better."**
Higher rank overfits and starts reproducing noise (and treated structure)
instead of $Y(0)$. Cross-validate the regularization; don't max the rank. §4.

**"It replaces SCM."**
It's the tool for *messy* panels. For one clean treated unit with a good
pre-period, SCM/SDID are more transparent and give interpretable weights.
Matrix completion earns its keep when the rollout is chaotic. §3.

## 6. Where this leaves the toolkit

With matrix completion, the panel/comparative-case family has an answer for
every rollout shape:

| Situation | Method |
|---|---|
| One treated unit, clean pre-trend | Classic SCM (§6.1) |
| Treated unit is a level outlier | SDID (§6.2) |
| Imperfect fit, modest extrapolation | ASCM (§6.2) |
| Staggered, absorbing | Callaway-Sant'Anna ([L3 §3.4](../03-difference-in-differences/3.3-staggered-did.md)) |
| **Messy: staggered + toggling + many units** | **Matrix Completion** |
