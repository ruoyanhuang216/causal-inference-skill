# 7.2 — Correlated Random Effects (The Mundlak Approach)

> **Part of:** [Lecture 7 — Panel Data Methods](./README.md)
> **Prerequisite:** [§7.1](./7.1-fixed-effects.md) — Mundlak recovers FE's β while curing FE's one big weakness.

---

## 1. FE's fatal flaw — it destroys static variables

Fixed effects (§7.1) is powerful, but it has a cost: **the within-transformation
subtracts anything that doesn't change over time.** A user's gender, signup
cohort, or OS is constant across all their days, so demeaning wipes it to zero
($1 - 1 = 0$). If the PM asks *"what's the baseline difference between iOS and
Android users?"*, FE literally cannot answer — it deleted the variable.

The classic alternative, **Random Effects (RE)**, *keeps* static variables but
assumes the unobserved $\alpha_i$ is **uncorrelated with the treatment** — an
assumption that is almost always false in industry (power users adopt *and*
engage), and which the **Hausman test** routinely rejects. So the naive menu is:
FE (unbiased β, no static coefficients) or RE (static coefficients, biased β).

**Mundlak (1978) gets both.**

---

## 2. The Mundlak hack

Instead of *assuming* $\alpha_i$ is uncorrelated with treatment, **model the
correlation directly.** Mundlak proposes that the unobserved effect is a linear
function of the unit's *average* treatment:

```math
\alpha_i = \gamma\,\bar X_i + u_i
```

Substitute into the panel model and you get a plain (random-effects) regression
with the group mean added as a control:

```math
Y_{it} = \beta X_{it} + \gamma\,\bar X_i + \delta Z_i + u_i + \epsilon_{it}
```

where $Z_i$ are the static variables (gender, OS). Operationally it's trivial:
**compute each unit's mean $\bar X_i$, add it as a new column, run a random-
effects model.** The $\bar X_i$ term soaks up the correlation between the
unobserved trait and the treatment — exactly the confounding FE removes by
demeaning.

---

## 3. The beautiful identity — Mundlak's β *is* FE's β

The payoff, and it's exact, not approximate: **the treatment coefficient $\beta$
from the Mundlak regression equals the fixed-effects estimate — to machine
precision** — while you *also* get valid, unbiased coefficients on the static
variables $Z_i$ that FE destroyed.

*Verified* (code in §5): with $\alpha_i$ correlated with $X$ and a true
$\beta = 1.5$, the FE within-estimator gives **1.504** and the Mundlak regression
gives **1.504** — identical to **1.3e-15.** Adding the group mean recovers FE's
confounder-scrubbing *and* keeps the static coefficients. It's FE without the
amnesia.

Why it works: $\bar X_i$ absorbs the *between*-unit variation (the part FE throws
away), leaving $\beta$ to be identified purely from *within*-unit variation — the
same variation FE uses. You've partitioned the treatment into its between- and
within-unit pieces and let $\bar X_i$ carry the confounded between-part.

---

## 4. Worked probe: Spotify, iOS vs. Android

**Setup (continuing §7.1).** Your FE model says push notifications cause a +5-min
lift. The PM: *"Great — but I also need the baseline behavioral difference between
iOS and Android users to allocate marketing budget."* FE destroyed the OS
variable (it doesn't change day to day).

**The Mundlak walkthrough:**

1. **Feature-engineer the Mundlak control.** For every user, compute their 30-day
   average notification state $\bar X_i$; add it as a static column
   `avg_notif_propensity`.
2. **Run a random-effects (mixed) model**, dropping the within-transformation.
3. **Include:** daily notification state $X_{it}$, the OS flag $Z_i$, and the
   Mundlak column $\bar X_i$.

**The result.** `avg_notif_propensity` soaks up the power-user bias, so the daily-
notification coefficient $\beta$ **exactly matches the unbiased FE estimate**,
*and* the model now reports a **valid coefficient for iOS vs. Android.** One model,
both answers — the causal within-effect *and* the descriptive static comparison
the business needs.

## 5. Python

```python
import numpy as np, statsmodels.api as sm

# ---- Mundlak: add the group mean of X, run a standard regression ----
Xbar = np.zeros(uid.max()+1); np.add.at(Xbar, uid, X); Xbar /= T   # per-unit mean
design = np.column_stack([X, Xbar[uid], Z])                        # X, Mundlak control, static Z
beta_mundlak = sm.OLS(Y, sm.add_constant(design)).fit(
    cov_type="cluster", cov_kwds={"groups": uid}).params[1]
# verified: beta_mundlak == beta_FE to 1e-15 (true 1.5, both 1.504)

# ---- The proper mixed-model version (random intercept per unit) ----
```

## 6. Common misreadings

**"Random effects and Mundlak are the same."**
RE assumes $\alpha_i \perp X$ (usually false). Mundlak *models* the correlation
via $\bar X_i$, so its β is unbiased and equals FE. §1–3.

**"Mundlak's β is approximately FE's β."**
It's *exactly* FE's β — verified to 1e-15. The group mean partitions treatment
into between/within; β is identified off the within part, same as FE. §3.

**"Use Mundlak to get the causal effect of gender."**
The static coefficients (gender, OS) are **descriptive/associational**, not
causal — you can't difference away confounders of a variable that never changes.
Mundlak recovers them for reporting, but don't call them causal. §4.

**"The Hausman test tells me to use RE."**
If Hausman rejects, RE is *biased* — you want FE or Mundlak, not RE. The Mundlak
regression is itself the cleanest way to run that test (jointly test the $\bar
X_i$ coefficients). §5.
