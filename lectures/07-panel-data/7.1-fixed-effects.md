# 7.1 — Fixed Effects (The "Within" Estimator)

> **Part of:** [Lecture 7 — Panel Data Methods](./README.md)

---

## 1. The philosophy — subtract each unit from itself

Users carry unobserved, time-invariant traits — intelligence, brand loyalty,
socioeconomic status — that a cross-sectional comparison can never isolate.
Fixed effects' move: **subtract each user's own historical average from their
daily data, and every stable trait cancels to zero.** What survives is pure
within-user variation.

Start from the panel model, where $\alpha_i$ is the unit's time-invariant
confounder:

```math
Y_{it} = \alpha_i + \beta X_{it} + \epsilon_{it}
```

**The within-transformation.** Average over time for unit $i$, then subtract:

```math
\bar Y_i = \alpha_i + \beta \bar X_i + \bar\epsilon_i \quad\Longrightarrow\quad (Y_{it} - \bar Y_i) = \beta(X_{it} - \bar X_i) + (\epsilon_{it} - \bar\epsilon_i)
```

$\alpha_i - \alpha_i = 0$. The confounder is **gone** — differenced away without
ever being measured. Regress the demeaned outcome on the demeaned treatment and
you get an unbiased $\beta$, no matter how badly $\alpha_i$ correlated with the
treatment.

*Verified* (code in §5): $\alpha_i$ deliberately correlated with $X$, true
$\beta = 1.5$. Pooled OLS returns **2.45** (badly biased by the confounding);
the FE within-estimator returns **1.504.**

---

## 2. When to use it

FE is the tool when a treatment **toggles on and off for the same unit over
time** — a premium subscription, a localized ad, a push-notification setting —
and you're terrified of unobserved baseline differences between users. You need
*within-unit* variation in the treatment: a unit that's always treated (or never)
contributes nothing, because it has no deviation from its own mean.

Equivalently, FE with **both** unit and time effects is the two-way-fixed-effects
(TWFE) estimator — which *is* difference-in-differences
([Lecture 3](../03-difference-in-differences/)). Panel FE and DiD are the same
machinery viewed from two angles.

---

## 3. Two things that will bite you

### 3.1 Cluster your standard errors — at the unit

You track the same user over many days, so $\epsilon_{i,t}$ is highly correlated
with $\epsilon_{i,t+1}$ (**serial correlation**). Use naive OLS standard errors
and your confidence intervals are **far too narrow** — you'll manufacture false
positives.

**Cluster the standard errors at the unit level** (`cluster="user_id"`). This is
the *exact same lesson* as [Lecture 3 §3.1](../03-difference-in-differences/3.1-classic-did-and-geo-controls.md#3-inference-the-standard-error-is-where-did-dies)
— because DiD is a panel FE method, its Bertrand-Duflo-Mullainathan
serial-correlation problem is *this* problem. The few-clusters caveats carry over
too: with few units, escalate to the wild cluster bootstrap.

### 3.2 Strict exogeneity — the assumption that quietly breaks FE

FE requires **strict exogeneity**: past *and future* treatments are uncorrelated
with the error at every period. The dangerous half is **feedback from past
outcomes to current treatment.** If a user has a bad experience Monday ($Y$) and
therefore cancels their subscription Tuesday ($X$), then $X_{t}$ depends on
$\epsilon_{t-1}$, strict exogeneity fails, and **FE is biased.**

This is the panel version of reverse causality, and it's especially lethal with a
**lagged dependent variable** ($Y_{t-1}$ as a regressor) — a violation so
systematic it has its own name and its own fix ([§7.3, Nickell bias](./7.3-dynamic-panel-gmm.md)).

## 4. Worked probe: Spotify push notifications

**Setup.** Measure the causal effect of turning on **push notifications** $X$ on
**daily minutes listened** $Y$.

**The confounder.** Users who *voluntarily* enable notifications are already power
users. Compare notifications-ON users to notifications-OFF users cross-
sectionally and you massively overstate the effect — you're measuring the
power-user trait, not the notification.

**The FE walkthrough.** Track 100,000 users over 30 days; some toggle
notifications on and off during the window.

1. For User A, compute their 30-day average listening (say 45 min/day) and
   average notification state (ON 30% of days).
2. Subtract A's 45-minute average from each day:
   - Day 1 (notifs OFF): listened 40 → transformed $Y = -5$.
   - Day 2 (notifs ON): listened 55 → transformed $Y = +10$.
3. Regress transformed $Y$ on transformed $X$.

**The result.** Subtracting the baseline **erased the power-user trait $\alpha_i$
entirely.** You are now asking only: *"on days User A shifted above their own
average notification state, did their listening shift above their own average?"*
— which is causal, because A is compared to A. The between-user selection that
wrecked the cross-sectional comparison is differenced out of existence.

**The catch to state:** this requires that a *bad listening day doesn't cause the
notification toggle* (§3.2). If users disable notifications *because* they've been
listening less, strict exogeneity fails and even FE is biased — you'd need a
design (Lecture 3/5) instead.

## 5. Python

```python
import numpy as np, statsmodels.api as sm

# Panel: uid (unit id), Y, X. T periods per unit.
def within(v, uid, T):
    m = np.zeros(uid.max()+1); np.add.at(m, uid, v); m /= T
    return v - m[uid]

# ---- FE via the within-transformation (no constant after demeaning) ----
beta_fe = sm.OLS(within(Y, uid, T), within(X, uid, T)).fit().params[0]
# verified: true 1.5, pooled OLS 2.45 (biased), FE 1.504
```

## 6. Common misreadings

**"FE controls for all confounding."**
Only *time-invariant* confounding. A confounder that moves with the treatment
(a motivation spike the week someone subscribes) is untouched. §1, §3.2.

**"I can estimate the effect of gender / signup cohort with FE."**
No — the within-transformation subtracts anything that doesn't change over time,
so it *destroys* static variables. That's what Mundlak (§7.2) fixes.

**"Standard OLS errors are fine after adding fixed effects."**
Serial correlation within a unit makes them far too narrow. Cluster at the unit
(same as DiD, §3.1). §3.1.

**"Always-treated and never-treated users still inform the estimate."**
They don't — FE uses *within-unit* variation, and a unit with no toggle has zero
deviation from its own mean. You identify off the switchers.

**"FE handles a lagged dependent variable."**
It doesn't — that's Nickell bias, and it's severe when $T$ is small. Use
Arellano-Bond (§7.3).
