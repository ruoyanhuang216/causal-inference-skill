# 9.3 — DML-IV (Partially Linear IV)

> **Part of:** [Lecture 9 — Double / Debiased Machine Learning](./README.md)
> **Prerequisites:** [§9.1](./9.1-dml-core.md) and [Lecture 5 §5.3 — Judge IV](../05-instrumental-variables/5.3-judge-examiner-iv.md). This chapter *merges* them.

This is where two threads of the series meet: the **Judge/Examiner IV** design
(Lecture 5) and **Double ML** (§9.1). When treatment is endogenous *and* the
confounders are high-dimensional, you need both — the IV to defeat unobserved
confounding, the ML to crush observed confounding and variance. The result is
**DML-IV**, implemented as the **Partially Linear IV (PLIV)** model. It's the
design that closes out the collection-agent problem from Lecture 5.

---

## 1. The collection-agent setup

Recall the variables (Lecture 5 §5.3):

- **$Y$ — outcome:** long-term metric (60-day payments).
- **$D$ — treatment:** kept amount (immediate money collected on the call).
- **$Z$ — instrument:** agent assignment, as the agent's **leave-one-out (LOO)
  effectiveness** — the mean kept amount they achieve on *all other* calls.
- **$X$ — confounders:** borrower demographics, loan history, credit scores.

There are **two different causal questions**, and they need two different DML
architectures. Naming which one you're answering is half the design.

---

## 2. Setup 1 — the agent's total effect (DML for the ITT)

**Question:** *"How much does assigning a borrower to an elite agent raise
60-day payments — regardless of how the agent achieves it?"*

This is the **ITT** (Lecture 1): the effect of the *assignment* $Z$, through
every channel. Because agent assignment is (conditionally) random, **you don't
need an instrument here** — you treat the agent as the treatment and run a
standard partially linear DML (§9.1):

- **Model Y:** predict $Y$ from $X$ → residual $\tilde Y$.
- **Model Z:** predict $Z$ from $X$ → residual $\tilde Z$. *(If routing is truly
  random, $X$ barely predicts $Z$, and $\tilde Z$ is essentially the random
  assignment itself — which is mathematically perfect.)*
- **Final stage:** OLS $\tilde Y$ on $\tilde Z$.

**Why DML rather than a simple A/B difference?** Because the ML Model Y
**absorbs the enormous variance in $Y$ driven by borrower background $X$.** The
agent's effect is a small signal on top of huge borrower heterogeneity;
residualizing $Y$ on $X$ strips that heterogeneity out, so the standard error on
the agent effect **shrinks dramatically.** Same estimand as the raw comparison,
far tighter confidence interval. This is variance reduction, the observational
cousin of CUPED ([Lecture 2 §2.1](../02-experimentation/2.1-rct-and-assumptions.md)).

---

## 3. Setup 2 — kept amount's effect (DML-IV / PLIV)

**Question:** *"For every marginal dollar an agent extracts on the call (kept
amount), how much does the 60-day payment rise?"*

Now the target is $D$ (kept amount), which is **endogenous**: borrowers who pay
today are generally more likely to pay in 60 days, regardless of the agent. So
you instrument $D$ with the agent effectiveness $Z$ — but you *also* have to
partial the 500 borrower features $X$ out of everything. That's **three**
cross-fitted nuisance models (versus two for the plain PLR):

- **Model Y:** predict $Y$ from $X$ → $\tilde Y = Y - \hat g(X)$.
- **Model D:** predict $D$ (kept amount) from $X$ → $\tilde D = D - \hat m(X)$.
- **Model Z:** predict $Z$ (agent instrument) from $X$ → $\tilde Z = Z - \hat r(X)$.

**Final stage — residual 2SLS.** Instead of OLS, run 2SLS on the residuals:
instrument the *unexplained kept amount* $\tilde D$ with the *unexplained agent
effectiveness* $\tilde Z$, to explain the *unexplained 60-day payment* $\tilde
Y$. For the just-identified case this is the Wald-style ratio:

```math
\hat{\theta}_{\text{PLIV}} = \frac{\tilde{Z} \cdot \tilde{Y}}{\tilde{Z} \cdot \tilde{D}}
```

This is exactly Lecture 5's IV logic — reduced form over first stage — but with
ML having partialled the borrower confounders out of all three quantities
first.

**What you identify:** because kept amount is *continuous*, this is an **ACR**
(Average Causal Response, Lecture 5 §6.2), local to borrowers whose kept amount
responds to agent skill — not always-payers or never-payers. Same estimand as
the Lecture 5 design, now robust to high-dimensional nonlinear confounding.

---

## 4. The architecture gotcha: many instruments

The trap in production (and the payoff of Lecture 5 §5.3):

**Don't use `agent_id` dummies as the instrument.** A 50-column one-hot vector
of agent identities is a **many-instruments** problem — the first stage overfits
those 50 columns, and the IV estimate **collapses back toward the biased OLS**
estimate. In Lecture 5 this was *verified*: 50 agent dummies put the 2SLS
estimate **19% of the way back to the OLS bias**; a single continuous LOO
instrument recovered the truth.

**The fix — LOO-DML:**

1. *Before* the DML pipeline, compute a **continuous leave-one-out effectiveness
   score** per agent: the mean kept amount that agent achieved on **all other**
   calls (excluding this borrower's, to break the mechanical correlation —
   Lecture 5 §5.3 §2).
2. Feed that **single continuous scalar** as the instrument $Z$.
3. DML's Model Z partials the borrower features $X$ out of that skill score,
   and the final residual-2SLS stage is stable.

Combining the **LOO Judge-IV** trick with **DML** gives you both: the IV defeats
the unobserved borrower psychology (the within-call channels, Lecture 5 §6.3),
and the ML cross-fitting crushes the observed high-dimensional confounding. It's
the state-of-the-art design for this problem.

## 5. Python

```python
import numpy as np
from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import KFold

def dml_pliv(Y, D, Z, X, K=5, seed=0):
    """Partially Linear IV via DML. Z instruments D; X partialled out of all three.
    Z should be the CONTINUOUS LOO agent score, not agent-id dummies (section 4)."""
    n = len(Y)
    Yt, Dt, Zt = (np.empty(n) for _ in range(3))
    ml = lambda: RandomForestRegressor(n_estimators=200, min_samples_leaf=5, n_jobs=-1)
    for tr, te in KFold(K, shuffle=True, random_state=seed).split(X):
        Yt[te] = Y[te] - ml().fit(X[tr], Y[tr]).predict(X[te])   # outcome residual
        Dt[te] = D[te] - ml().fit(X[tr], D[tr]).predict(X[te])   # treatment residual
        Zt[te] = Z[te] - ml().fit(X[tr], Z[tr]).predict(X[te])   # instrument residual
    theta = (Zt @ Yt) / (Zt @ Dt)                                # residual 2SLS (Wald)
    # first-stage strength on residuals -> report it (weak-instrument check, Lecture 5)
    F = ((Zt @ Dt) ** 2 / (Zt @ Zt)) / ((Dt - (Zt @ Dt)/(Zt @ Zt) * Zt) ** 2).mean()
    return theta, F

# ---- Setup 1 (ITT) is just dml_plr(Y, Z, X) from 9.1 -- residualize Y and Z on X. ----
# ---- Build the LOO instrument BEFORE the pipeline: ----
#   agent_mean = df.groupby('agent')['kept'].transform('sum')
#   Z = (agent_mean - df['kept']) / (df.groupby('agent')['kept'].transform('count') - 1)
# ---- Production: econml.iv.dml.DMLIV / OrthoIV, or doubleml.DoubleMLPLIV ----
```

## 6. Common misreadings

**"Use DML-IV for the agent's total effect."**
The agent's *total* effect (ITT) needs no instrument — assignment is random, so
it's a plain PLR (Setup 1). DML-IV is for the *kept-amount channel* (Setup 2).
§2 vs §3.

**"Feed the 50 agent dummies in as the instrument."**
Many-instruments bias pulls the estimate back toward OLS (verified 19% in
Lecture 5). Collapse to a single continuous LOO score. §4.

**"PLIV with a continuous treatment gives the LATE."**
Continuous $D$ gives the **ACR** — the effect per marginal dollar, for
responsive borrowers. Name it correctly. §3.

**"DML-IV removes the need for the exclusion restriction."**
No. It removes high-dimensional *observed* confounding, but the instrument still
needs exclusion (agent affects $Y$ only through kept amount) and monotonicity —
the within-call-channel threat from Lecture 5 §6.3 is untouched by ML. §3.

**"Two nuisance models are enough for the IV version."**
PLIV needs **three** — Y, D, *and* Z all residualized on $X$. Skipping Model Z
leaves borrower confounding in the instrument. §3.
